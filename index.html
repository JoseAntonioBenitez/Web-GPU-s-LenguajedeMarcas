What is it, what does it do and what is it for?

A graphics processing unit (GPU) is a coprocessor dedicated to graphics processing or floating point operations, to lighten the workload of the central processor/CPU in applications such as video games or interactive 3D applications. Thus, while much of the graphics-related work is processed on the GPU, the central processing unit (CPU) can be dedicated to other types of calculations (such as artificial intelligence or mechanical calculations in the case of video games).

The GPU implements certain graphics operations called primitives optimized for graphics processing. One of the most common primitives for 3D graphics processing is anti-aliasing, which smoothes the edges of figures to give them a more realistic appearance. Additionally there are primitives for drawing rectangles, triangles, circles and arcs. GPUs currently have a large number of primitives, seeking greater realism in the effects).

GPUs are present in graphics cards.

Types of GPU

Integrated cards

These can be found both integrated in the board and in the processor itself, which is why they use the system memory to develop their operation. They have a section exclusively dedicated to graphics, it has a different architecture, designed exclusively to generate these pixel images.

Dedicated cards

This type of graphic unit has a series of specific functions assigned to it and is designed to expressly fulfill them. Its main characteristic is that it has a RAM memory independent from that of the computer.

The dedicated graphics card is on a daughterboard (put a picture explaining parts of the graphics, VRM, where the power comes from etc).

How does it work?

The processor sends it the order to generate an image, and this is responsible for taking out the pixel matrix that sends through the HDMI, VGA, DVI or Displayport cable to the monitor at a specific frequency (usually 60 images per second).

When we want to do something more like playing video games, in which we have polygonal images that are transformed into flat images and displayed on the screen, or for example special 3D design programs, special effects or video editing that are things that will need a little more power than we can give us the integrated graphics we have in the processor.

GPU Implemented Technologies

The Shader technology


Shaders are a special type of program that runs on graphics processing units (GPUs).



The output of the shader is the colored pixels of the object drawn in the viewport. Some shaders allow specialized outputs (especially with APIs like Vulkan). The standard process is the "vertex -> frgament" shader channel. The vertex shader is used to calculate where each vertex (a point in a 3D model) is located and the fragment shader calculates the color of the individual pixels.

AntiLiasing Technology

Anti-Aliasing is nothing more than a post-processing filter, and therefore performed after generating the image, which smoothes these sawtooths so that lines that are not vertical or horizontal appear more homogeneous, eliminating or at least mitigating this step effect.

If we have a high-end computer, with a good graphics card, we can use SSAA or MSAA as antialiasing. These are the most widespread and the ones that will give us the best quality in the graphics that we have closer to the screen.

Raytraicing technology or RTX

Ray tracing is a technology developed by NVIDIA that is responsible for improving the lighting, shadows and reflections of video games. In English, ray tracing means "ray tracing" and refers to a much more precise calculation of in-game lighting that impacts reflections and shadows.
With this algorithm, the scenes look much more realistic, which is why it has been a revolution for the gamer community.

NVIDIA began developing the technology several years ago, but it wasn't until 2018 that its first cards with ray tracing came out.It's very easy to recognize the graphics cards that have this algorithm because the company changed the nomenclature so that now its acronym is RTX.



Sli vs CrossFire

These are two tools that were created for use with NVIDIA and AMD respectively and have their origins in the multiGPU configurations used with Voodoo 2.

SLI, which stands for Scan Line Interleave, works in a simple way, using two cards in parallel with a 50/50 workload balance that even allows splitting the screen in two.

It is a bit of an old tool, which required doubling the rendering power, leaving out the memory for textures because each graphics card requires duplicating the data of the space in which it is performing.
SLI mode is currently used by NVIDIA graphics solutions, while CrossFire is used by AMD as mentioned above. Specifically, both terms refer to a mode of use of two or more graphics cards that are placed to work together and simultaneously on the same computer or system. Currently these technologies are almost in disuse because now the graphics cards are so powerful that it is not necessary to use it, plus many programs and games no longer implement this technology.

Where we can see that it is still being used is in cryptocurrency mining stations.



Differences between the core of a GPU and core of a CPU

The main difference is that CPUs are primarily designed for instruction-level parallelism, while GPUs specialize in thread-level parallelism.

Instruction-level parallelism seeks to reduce the instruction time of a program by executing several program instructions simultaneously. Kernels based on thread-level parallelism take several programs at the same time and execute them in parallel.

What are the GPU components?

There are a number of common elements used by both NVIDIA GPUs and AMD GPUs that are essential for a GPU to do its job.

SM or Compute Units:
NVIDIA uses the former, AMD the latter. Each unit integrates a set of shader engines, texture units, raster units and, depending on the architecture, specialized cores for ray tracing and artificial intelligence. They are the foundation of any GPU.

Shader engines:
Also known as shaders. They are responsible for performing the workload associated with tasks as important as the transformation of geometry, both at the level of color and shading and other effects (lighting, haze, reflections, etc.).

Raster units:
Better known as ROPs. They perform the process necessary for an image to be converted into a set of perfectly ordered pixels that are written to the framebuffer, from which they will be transmitted directly to the screen. They also perform important operations associated with the application of filters and smoothing techniques (MSAA, for example).

Texturizing units:
These are concerned with applying texture mapping to the geometry.

RT cores:
Dedicated to speed up the processing of ray tracing related workload. These kernels were introduced in 2018, and focus on calculating BVH transverse intersections, ray-delta intersections and to frame delimiter intersections.

Tensor cores:
These cores are also relatively new. They are only used in NVIDIA graphics cards based on the Volta, Turing and Ampere architectures. They are specialized for workloads focused on inference, artificial intelligence and deep learning. In gaming, they handle the workload of applying DLSS 2.0.

Memory bus:
The memory bus determines the width available for the GPU to communicate with the graphics memory.

Graphic memory:
It is not integrated into the GPU, but it plays a very important role. Its function is, in essence, very similar to that of RAM with respect to the CPU. The GPU uses VRAM to store certain graphic elements and data that it has already processed, so that it can call them up when it needs them without having to process them again. An insufficient amount of graphics memory can limit the performance of a GPU, and its operating frequency also has an impact, as slower graphics memory offers lower access (write and read) speeds.
